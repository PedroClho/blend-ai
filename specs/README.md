# Specs

Feature não-trivial começa por uma spec curta aqui **antes** de codar (spec-driven — ver `CLAUDE.md`, Fluxo de desenvolvimento).

Template (`specs/<nome>.md`):

- **Problema / motivação**
- **Hipótese** (se for experimento) e como validar
- **Escopo** — o que entra / o que não entra
- **Interface** com outros módulos (tipos em `src/blend/types.py`)
- **Critério de pronto** — incl. testes em `tests/`
